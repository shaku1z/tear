# Failure interpretation

## Architecture, type, and lint

Treat these as ownership or dependency-direction failures. Inspect `docs/ARCHITECTURE.md`, typed ports, imports, forbidden suppressions, production JavaScript, and subsystem size. Do not move browser/platform behavior inward or suppress the error.

## Unit, contract, and conformance

Identify whether the failed assertion protects a definition catalogue, deterministic rule, state transition, serialization envelope, adapter contract, or compatibility behavior. Prefer correcting the implementation; update an assertion only when the intended contract changed and its neighboring evidence is updated consistently.

## Browser and journeys

These execute production artifacts. Separate build staleness from product failure first. Then inspect console/page errors, canonical debug preparation, real input routing, screen transitions, viewport mapping, lifecycle behavior, and terminal restoration. Do not bypass production actions to make a journey pass.

## Audio and provenance

Distinguish authored integration defects from vendored-input provenance drift. Keep one host-owned audio context, exclusive music backends, semantic routing, mixer hierarchy, lifecycle cleanup, and verified TearScore/Tone inputs.

## Platform and CrazyGames

Check typed capability availability, explicit unavailable fallbacks, SDK initialization, loading/gameplay brackets, ads, temporary mute reasons, storage mirroring, relative asset paths, iframe behavior, and package contents. Gameplay must remain host-agnostic.

## Bundles and reproducibility

Bundle failures protect size/file budgets. Reproducibility failures protect byte-stable builds and target isolation. Look for nondeterministic output, leaked source/tests/repository files, opposite-target integrations, or absolute asset paths.

## Cloudflare and PWA

Cloudflare must upload only `dist/standalone`. PWA behavior is generated by Vite/Workbox and must prove offline boot and safe update recovery. Never reintroduce manual cache versions or service-worker asset lists.

## Release gate

`pnpm check` composes all release evidence. Report the first failing stage and any later stages that did not run; do not describe an incomplete chain as a full pass.
